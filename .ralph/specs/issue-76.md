# Certify the accessible bilingual core journey

> GitHub issue #76 | Labels: ready-for-agent, P1 | https://github.com/taigamura/simple-bookkeeping/issues/76

## Parent

#72

## What to build

Certify one complete core ledger journey as accessible and coherent in both Japanese and English. Audit the real Calendar, Summary, Entry, Settings, Budgets, authentication, destructive-action, and empty-state flows; fix release-blocking labels, roles, states, focus order, touch targets, scaling, contrast, or untranslated/misleading copy without redesigning the visual system.

## Acceptance criteria

- [ ] Every core control and status has an accurate accessible role, label, value/state, disabled state, and logical navigation order in Japanese and English.
- [ ] Core flows remain usable with VoiceOver and supported larger text settings; essential controls are not clipped, hidden, or dependent on color alone.
- [ ] Release-facing copy contains no unintended language fallback and accurately describes delete-all scope, empty first launch, symbol-only currency formatting, and materialize-on-save recurrence.
- [ ] Touch targets and contrast meet the existing design's accessibility intent without a broad visual redesign.
- [ ] Automated component assertions cover observable accessibility metadata, and a concise native VoiceOver/text-size checklist records the remaining device verification.
