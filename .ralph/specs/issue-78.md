# Prepare the bilingual App Store publication package

> GitHub issue #78 | Labels: ready-for-agent, P1 | https://github.com/taigamura/simple-bookkeeping/issues/78

## Parent

#72

## What to build

Prepare a complete, name-ready Japanese and English App Store publication package that accurately presents the certified product and privacy mode. Produce final-form listing copy with an explicit product-name placeholder, a non-sensitive screenshot plan and capture dataset, privacy/support/review information, and a field-by-field App Store checklist so applying the later naming decision is mechanical rather than a new content project.

## Acceptance criteria

- [ ] Japanese and English drafts cover name placeholder, subtitle, description, keywords, promotional copy if used, Finance category, age rating, copyright, support details, privacy-policy URL, export compliance, and review notes.
- [ ] Listing copy clearly explains the local-only design, absence of login/accounts/sync/bank connections, Zaim-compatible CSV import/export, symbol-only currency behavior, and materialize-on-save recurrence without overstating capabilities.
- [ ] A screenshot matrix covers every required locale and App Store device class using current UI, fictitious non-sensitive data, and no prototype or unshipped feature.
- [ ] Privacy answers match the certified zero-data-collection mode and the public English/Japanese privacy policy and support URLs are verified from a logged-out browser.
- [ ] Review notes tell App Review that there is no account or login and explain where to find CSV import/export and optional device authentication.
- [ ] The package identifies every place where the final public name must be substituted and can be completed without reopening product-copy decisions.
